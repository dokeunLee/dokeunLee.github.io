## 발표 개요

- **언제 / 어디서:** 2025년 / 대한전자공학회 추계학술대회
- **발표 논문:** 자율주행 로봇의 센서 데이터 처리 SLM 모델 연구
- **논문집:** 2025년 대한전자공학회 추계학술대회 논문집
- **저자:** 권형준, 박서연, 이동현, 이도근, 김동균, 오재덕
- **주제:** 자율주행 로봇의 IMU 센서 데이터를 분석하고, 이를 바탕으로 SLM 기반 보고서 초안을 생성하는 파이프라인 구현

이번 연구는 자율주행 로봇에서 수집되는 IMU 센서 데이터를 단순히 저장하거나 시각화하는 데 그치지 않고, 전처리·특징 추출·상태 군집화·보고서 생성까지 연결하는 전체 파이프라인을 구현하는 것을 목표로 했습니다. 특히 센서 데이터 분석 결과를 연구자가 직접 해석하고 문서화하는 데 드는 시간을 줄이기 위해, Small Language Model 기반의 보고서 초안 생성 구조를 설계했습니다.

## 연구 핵심 내용

본 연구에서는 자율주행 로봇의 IMU 시계열 데이터를 대상으로 전처리와 특징 추출을 수행했습니다. 이상치 제거에는 IsolationForest를 활용했고, 데이터의 노이즈를 줄이기 위해 Kalman Filter를 적용했습니다. 이후 Sliding Window 방식으로 데이터를 나누고, RMS와 FFT 기반 특징을 추출하여 로봇의 움직임 특성을 분석했습니다.

추출된 특징은 PCA를 통해 주요 성분으로 축소한 뒤 K-Means Clustering을 적용하여 로봇의 상태를 3개 그룹으로 분류했습니다. 이를 통해 정지 또는 저가속 상태, 가속 상태, 급격한 움직임 또는 비정상 구간을 구분하고, SLM이 보고서를 생성할 때 활용할 수 있는 구조화된 입력 데이터로 변환했습니다.

또한 보고서 생성을 위한 학습 데이터셋을 구축하기 위해 YOLOv11 기반의 문서 제목 영역 탐지 모델과 EasyOCR을 활용했습니다. 보고서 이미지에서 대제목, 중제목, 소제목을 탐지하고 OCR로 본문을 추출한 뒤, 문서의 논리적 구조를 JSON 형태로 재구성했습니다.

최종적으로 Polyglot-ko-5.8b 모델에 QLoRA 기반 파인튜닝을 적용하여 경량화된 SLM 모델을 구성했습니다. 이를 통해 엣지 디바이스 환경에서도 활용 가능한 보고서 생성 모델의 가능성을 확인하고자 했습니다.

## 준비 과정

이번 논문을 준비하면서 가장 중요하게 본 부분은 단순히 모델 하나를 학습시키는 것이 아니라, 센서 데이터가 보고서 형태의 결과물로 이어지는 과정을 하나의 시스템으로 구성하는 것이었습니다.

준비 과정은 크게 다음 흐름으로 정리할 수 있습니다.

1. IMU 센서 데이터 전처리
2. RMS, FFT 기반 특징 추출
3. PCA와 K-Means를 활용한 상태 군집화
4. YOLOv11과 OCR을 활용한 보고서 구조 데이터셋 구축
5. QLoRA 기반 SLM 파인튜닝
6. React와 Rust 기반 보고서 수정 인터페이스 구현

발표 자료를 구성할 때는 기술 요소를 나열하는 방식보다, “센서 데이터가 어떻게 보고서 초안으로 변환되는가”라는 흐름이 보이도록 정리하는 데 집중했습니다. 특히 IMU 데이터 분석 파이프라인과 문서 구조화 파이프라인이 서로 다른 기술처럼 보이지 않도록, 최종 목표인 자동 보고서 생성과 연결해 설명하려고 했습니다.

## 예상 질문 & 답변

| 질문 | 답변할 내용 | 보완할 점 |
|------|-------------|-----------|
| 왜 LLM이 아니라 SLM을 사용했는가? | 자율주행 로봇과 엣지 디바이스 환경에서는 연산 자원과 응답 속도가 중요하기 때문에, 경량화가 가능한 SLM 구조가 더 적합하다고 설명할 수 있습니다. | SLM을 선택한 정량적 근거, 예를 들어 추론 속도나 메모리 사용량 비교가 추가되면 더 설득력 있을 것입니다. |
| IMU 데이터만으로 로봇 상태를 충분히 설명할 수 있는가? | IMU는 로봇의 가속도와 각속도를 측정하기 때문에 움직임 변화 분석에는 유효하지만, 환경 인식까지 설명하기에는 한계가 있습니다. | 향후 LiDAR, 카메라, GPS 등 멀티모달 센서와 결합하는 방향을 더 구체화할 필요가 있습니다. |
| SLM 생성 결과의 품질은 어느 정도인가? | 현재 모델은 보고서 형식과 일부 문맥을 생성할 수 있지만, 반복 문장과 환각 현상이 있어 초기 단계입니다. | 생성 결과 평가 지표나 사람 평가 기준을 추가하면 연구 완성도가 더 높아질 수 있습니다. |
| YOLO와 OCR을 사용한 이유는 무엇인가? | 보고서의 제목 구조와 본문을 자동으로 분리하여 SLM 학습 데이터셋을 구축하기 위해 사용했습니다. | 문서 구조 추출 성능이 SLM 학습 품질에 미치는 영향을 더 자세히 분석할 필요가 있습니다. |

## 배운 점 / 다음에 적용할 것

이번 연구를 통해 AI 모델의 성능만큼이나 데이터 구조화 과정이 중요하다는 점을 확인했습니다. 특히 SLM이 보고서를 잘 생성하기 위해서는 단순한 텍스트 데이터보다, 제목·본문·섹션 구조가 명확히 정리된 학습 데이터가 필요했습니다.

또한 센서 데이터 기반 AI 시스템에서는 모델 자체보다 입력 데이터의 품질과 전처리 방식이 결과에 큰 영향을 미친다는 점을 체감했습니다. IMU 데이터의 노이즈 제거, 특징 추출, 군집화 과정이 제대로 설계되어야 SLM이 해석 가능한 입력을 받을 수 있었습니다.

다음 연구나 발표에서는 다음 부분을 보완하고자 합니다.

- SLM 생성 결과에 대한 정량적 평가 지표 추가
- IMU 외에 카메라, LiDAR, GPS 등 멀티모달 센서 데이터 확장
- QLoRA 적용 전후의 메모리 사용량 및 추론 속도 비교
- 문서 구조 데이터셋의 품질 개선
- 실제 로봇 주행 상황에서 생성된 보고서의 활용성 평가
