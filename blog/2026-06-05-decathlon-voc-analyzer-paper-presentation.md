## 발표 개요

- **언제 / 어디서:** 2026년 6월 5일 예정 / 한국정보기술학회 학술대회
- **발표 논문:** Decathlon VOC Analyzer: 상품 이미지·텍스트 증거와 사용자 리뷰 정렬을 위한 증거 기반 멀티모달 VOC 분석 시스템
- **영문 제목:** Decathlon VOC Analyzer: An Evidence-driven Multimodal VOC Analysis System for Aligning Product Image-Text Evidence with User Reviews
- **저자:** 예보타오, 이도근, 류우솽, 정서완, 정현준, 전혜, 김재수†
- **주제:** 상품 이미지·텍스트 증거와 사용자 리뷰를 정렬하여, 전자상거래 VOC 분석 결과를 근거 기반 구조화 보고서로 생성하는 멀티모달 분석 시스템 구현

이번 논문은 전자상거래 환경에서 사용자 리뷰에 담긴 주관적 경험과 상품 페이지에 존재하는 객관적 정보를 함께 분석하는 것을 목표로 했습니다. 단순히 리뷰를 요약하는 방식이 아니라, 상품 이미지·텍스트·리뷰를 하나의 증거 공간으로 구성하고, 각 리뷰 측면이 어떤 상품 증거에 의해 지지되는지 추적 가능한 형태로 연결하는 데 초점을 두었습니다.

## 연구 핵심 내용

본 연구에서는 Decathlon 상품 데이터를 대상으로 상품 텍스트, 이미지, 이미지 영역, 사용자 리뷰를 통합적인 증거 패키지로 표준화했습니다. 이후 리뷰에서 속성, 감성, 의견, 원문 근거, 사용 장면을 포함하는 측면 수준의 VOC 객체를 추출하고, 각 측면을 텍스트 지지, 시각 확인, 교차 모달 설명을 위한 질문으로 변환했습니다.

핵심 구조는 크게 네 계층으로 구성됩니다.

1. **상품 증거 계층:** 상품명, 설명, 규격, 이미지, 리뷰를 재사용 가능한 증거 객체로 보존
2. **VOC 요구 모델링 계층:** 사용자 리뷰를 필터링하고 측면 수준 VOC 객체로 변환
3. **질문 유도 멀티모달 검색 계층:** 리뷰 측면을 증거 검색을 위한 질문으로 분해하고 텍스트·이미지 경로에서 후보 증거 검색
4. **보고서 계층:** 강점, 약점, 개선 제안, 증거 공백을 claim 단위로 생성하고 각 claim을 리뷰·상품 텍스트·상품 이미지 증거에 귀속

이 구조의 핵심은 리뷰 문장을 그대로 검색 질의로 사용하지 않는다는 점입니다. 리뷰 표현은 사용자마다 다르고 문맥 의존성이 크기 때문에, 시스템은 먼저 리뷰 측면을 명확한 증거 의도로 분해합니다. 예를 들어 크기 불만에 관한 리뷰는 상품 설명과 규격에서 텍스트 근거가 있는지, 이미지에서 관련 구조를 확인할 수 있는지, 텍스트와 이미지가 동일한 해석을 제공하는지를 각각 묻는 방식으로 처리됩니다.

## 증거 점수와 claim 귀속

제안 시스템은 리뷰 측면과 후보 증거의 관련성을 통합적으로 평가하기 위해 텍스트 관련성, 이미지 관련성, 교차 모달 일관성을 하나의 증거 점수로 결합했습니다.

```text
S(a,e) = λₜRₜ(a,e) + λᵥRᵥ(a,e) + λ꜀C(a,e)
```

여기서 `Rₜ(a,e)`는 텍스트 경로의 관련성, `Rᵥ(a,e)`는 이미지 경로의 관련성, `C(a,e)`는 리뷰 측면과 상품 증거 사이의 교차 모달 일관성을 의미합니다. 이 점수는 최종 판단을 대체하기 위한 것이 아니라, 검색과 재순위 단계에서 후보 증거를 일관된 기준으로 선택하기 위한 장치로 활용됩니다.

최종 보고서에서는 단순 분석 결과만 제시하지 않고, 각 claim이 어떤 리뷰 증거, 상품 텍스트 증거, 상품 이미지 증거에 의해 지지되는지를 함께 보존합니다. 이를 통해 보고서 결과가 단순 생성물이 아니라 검토 가능한 근거 기반 산출물이 되도록 설계했습니다.

## 구현 및 평가

현재 구현은 데이터셋 개요 확인, 상품 데이터 정규화, 인덱스 구축, 리뷰 측면 추출, 단일 상품 종합 분석을 위한 API를 포함합니다. 또한 전체 워크플로 실행, 오프라인 실행, 멀티모달 실행 검증, manifest 평가, 테스트 실행을 위한 스크립트를 제공하여 실험 재현성과 오류 분석이 가능하도록 구성했습니다.

평가 관점에서는 검색 품질과 귀속 품질을 분리했습니다. 검색 품질은 질문 수준의 관련 증거 주석이 있을 경우 Recall@K, MRR, NDCG와 같은 지표로 평가할 수 있으며, 구조화 보고서의 claim은 citation precision, contradiction rate, modality contribution과 같은 지표로 검토할 수 있습니다.

현재 단계에서는 대규모 수작업 주석 benchmark를 통한 최종 성능 경쟁보다는, 증거 기반 VOC 분석 프로토타입이 안정적으로 실행되는지를 검증하는 데 초점을 두었습니다. 구현 결과 주요 워크플로와 인터페이스의 일관성을 확인하기 위해 166개의 자동화 테스트를 통과했습니다.

## 발표 준비 포인트

이번 발표에서는 기술 요소를 단순히 나열하기보다, “리뷰 의견이 어떻게 검증 가능한 상품 증거와 연결되는가”라는 흐름을 중심으로 설명할 예정입니다. 특히 다음 세 가지를 핵심 메시지로 정리하려고 합니다.

1. VOC 분석은 단순 리뷰 요약이 아니라, 리뷰 측면과 상품 증거를 연결하는 문제로 보아야 한다.
2. 상품 페이지의 텍스트와 이미지를 함께 활용해야 리뷰의 주관적 의견을 더 정확하게 해석할 수 있다.
3. 생성형 AI 기반 보고서는 claim 단위 근거 귀속이 가능해야 실무 의사결정에 활용될 수 있다.

## 예상 질문 & 답변

| 질문 | 답변할 내용 | 보완할 점 |
|------|-------------|-----------|
| 기존 리뷰 요약 시스템과 무엇이 다른가? | 단순히 리뷰를 요약하는 것이 아니라, 리뷰 측면을 상품 텍스트·이미지 증거와 정렬하고 claim 단위로 근거를 귀속한다는 점이 차별점입니다. | 기존 VOC 분석 또는 리뷰 마이닝 시스템과의 비교 실험을 추가하면 더 설득력 있습니다. |
| 왜 멀티모달 접근이 필요한가? | 전자상거래 상품 정보는 텍스트 설명만으로 충분하지 않고, 크기, 형태, 색상, 구조처럼 이미지에서 확인해야 하는 정보가 많기 때문입니다. | 이미지 evidence가 실제로 보고서 품질을 얼마나 개선하는지 정량 분석이 필요합니다. |
| 질문 계획 계층을 둔 이유는 무엇인가? | 원시 리뷰 문장은 표현이 불안정하기 때문에, 텍스트 지지·시각 확인·교차 모달 설명처럼 검색 의도가 명확한 질문으로 변환해야 증거 검색의 추적성이 높아집니다. | 질문 생성 품질을 평가하는 별도 기준이 있으면 연구 완성도가 높아집니다. |
| 166개 테스트는 무엇을 의미하는가? | 전체 워크플로, 주요 API, 중간 산출물 보존, 멀티모달 실행 검증 등 시스템이 안정적으로 작동하는지를 확인한 자동화 테스트입니다. | 테스트 통과가 분석 정확도를 직접 보장하는 것은 아니므로, 향후 benchmark 기반 성능 평가가 필요합니다. |
| 실제 기업 의사결정에 어떻게 활용할 수 있는가? | 상품별 강점, 약점, 개선 제안, 증거 공백을 구조화해 제공하므로 제품 개선, 상세페이지 수정, 고객 불만 대응 우선순위 설정에 활용할 수 있습니다. | 실제 운영 데이터와 연결한 사용자 평가 또는 현업 피드백 검증이 추가되면 좋습니다. |

## 배운 점 / 다음에 적용할 것

이번 연구를 통해 생성형 AI 시스템에서 중요한 것은 단순한 답변 생성 능력만이 아니라, 답변이 어떤 데이터와 증거를 기반으로 만들어졌는지를 추적할 수 있는 구조라는 점을 확인했습니다. 특히 VOC 분석처럼 기업 의사결정과 연결되는 영역에서는 생성 결과의 자연스러움보다 근거성, 재현성, 오류 분석 가능성이 더 중요하다고 볼 수 있습니다.

다음 연구나 발표에서는 다음 부분을 보완하고자 합니다.

- 여러 상품 카테고리에 대한 수작업 주석 benchmark 구축
- 텍스트 검색, 이미지 검색, 멀티모달 검색 전략의 비교 실험
- claim 수준 citation precision 및 contradiction rate 평가
- 상품 이미지 내 세부 영역 grounding 성능 개선
- 사람 피드백 기반 replay 메커니즘 강화
- 실제 기업 VOC 분석 업무에서의 활용성 검증
