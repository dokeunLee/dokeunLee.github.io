# 2장. 머신러닝의 핵심과 지도학습

## 1. 머신러닝의 핵심

머신러닝은 사람이 규칙을 일일이 넣는 대신, **데이터로부터 패턴을 학습하는 데이터 기반 방법론**이다.

전통적인 프로그래밍에서는 사람이 규칙을 설계한다.

```text
규칙 + 입력 → 출력
```

반면 머신러닝에서는 데이터로부터 규칙에 해당하는 패턴을 학습한다.

```text
입력 + 출력 데이터 → 학습된 모델
```

즉, 머신러닝의 핵심은 데이터 속에 숨어 있는 관계를 모델이 학습하도록 만드는 것이다.

---

## 2. 지도학습과 비지도학습

머신러닝은 크게 지도학습과 비지도학습으로 나눌 수 있다.

| 구분 | 의미 | 예시 |
|---|---|---|
| 지도학습 | 정답, 즉 레이블이 있는 데이터로 학습 | 가격 예측, 스팸 메일 분류 |
| 비지도학습 | 정답 없이 데이터의 구조나 패턴을 찾음 | 군집화, 차원 축소 |

이 수업은 주로 **지도학습**에 초점을 둔다.

지도학습에서는 입력과 정답이 함께 주어진다.

```text
입력 x → 정답 y
```

모델은 학습 데이터를 이용해 x와 y 사이의 관계를 찾고, 새로운 입력이 들어왔을 때 결과를 예측한다.

---

## 3. 지도학습의 4단계

지도학습은 대략 다음 네 단계로 진행된다.

### 1. 모델 형태를 정한다

먼저 어떤 함수 형태를 사용할지 정한다.

예를 들어 선형회귀에서는 입력과 출력 사이의 관계를 직선 형태로 가정한다.

```text
y ≈ β₀ + β₁x
```

즉, 모델 형태를 정한다는 것은 데이터를 설명할 함수의 구조를 선택하는 것이다.

### 2. 목표를 정한다

다음으로 모델이 무엇을 잘해야 하는지 기준을 정한다.

보통은 예측값과 실제값의 차이를 줄이는 방향으로 목표를 설정한다.

회귀 문제에서는 평균제곱오차, 즉 MSE 같은 지표를 사용할 수 있다.

### 3. 학습 데이터를 이용해 파라미터를 찾는다

모델 구조와 목표가 정해지면, 학습 데이터를 이용해 최적의 파라미터를 찾는다.

이 과정이 학습이다.

모델은 손실 함수를 줄이는 방향으로 파라미터를 조정한다.

### 4. 새로운 데이터에 대해 예측한다

학습이 끝난 모델은 새로운 입력 데이터에 대해 예측을 수행한다.

중요한 점은 학습 데이터만 잘 맞추는 것이 아니라, 보지 못한 데이터에서도 잘 작동해야 한다는 것이다.

---

## 4. 중요한 용어

지도학습에서 자주 쓰는 용어는 다음과 같다.

| 용어 | 의미 |
|---|---|
| Target / Response | 예측해야 하는 대상 변수 |
| Feature / Input / Predictor | 예측에 사용하는 입력 변수 |
| Error | 실제값과 모델 예측값 사이의 차이 |
| Parameter | 모델이 데이터로부터 학습하는 값 |

현실 데이터에는 항상 오차가 존재한다.

따라서 모델의 목표는 오차를 완전히 없애는 것이 아니라, 데이터를 잘 설명하면서 새로운 데이터에 대해서도 일반화되는 관계를 찾는 것이다.

---

## 5. 파라메트릭 모델과 비파라메트릭 모델

모델은 크게 파라메트릭 모델과 비파라메트릭 모델로 나눌 수 있다.

| 구분 | 특징 |
|---|---|
| 파라메트릭 모델 | 구조가 비교적 단순하고 해석이 쉬움 |
| 비파라메트릭 모델 | 더 유연하지만 복잡해질 수 있음 |

파라메트릭 모델은 함수의 형태를 어느 정도 정해놓고 파라미터를 학습한다.

반면 비파라메트릭 모델은 더 유연하게 데이터의 구조를 따라갈 수 있지만, 복잡도가 커질 수 있다.

---

## 6. 성능 평가와 일반화

회귀 문제에서는 성능 평가에 MSE, 즉 평균제곱오차 같은 지표를 사용할 수 있다.

하지만 중요한 것은 학습 데이터를 잘 맞추는 것이 아니다.

핵심은 **새로운 데이터에서도 잘 작동하는 일반화 성능**이다.

그래서 보통 데이터를 다음처럼 나누어 평가한다.

```text
훈련 데이터
테스트 데이터
```

훈련 데이터는 모델을 학습하는 데 사용하고, 테스트 데이터는 학습된 모델이 새로운 데이터에 대해 얼마나 잘 작동하는지 평가하는 데 사용한다.

---

## 7. 과적합과 과소적합

모델이 너무 복잡하면 학습 데이터를 지나치게 잘 외우게 된다.

이것을 과적합이라고 한다.

반대로 모델이 너무 단순하면 데이터의 중요한 패턴을 제대로 잡지 못한다.

이것을 과소적합이라고 한다.

| 문제 | 의미 |
|---|---|
| Overfitting | 모델이 너무 복잡해서 훈련 데이터에 과하게 맞음 |
| Underfitting | 모델이 너무 단순해서 데이터 구조를 충분히 설명하지 못함 |

좋은 모델은 훈련 데이터뿐 아니라 테스트 데이터에서도 안정적으로 성능을 내야 한다.

따라서 성능뿐 아니라 해석 가능성도 고려해야 하며, 보통은 간단한 모델부터 시작하는 것이 바람직하다.

이와 관련해 **바이어스-베리언스 트레이드오프** 개념도 중요하다.

---

## 8. 회귀와 분류

머신러닝 문제는 크게 회귀와 분류로 나눌 수 있다.

| 구분 | 의미 | 예시 |
|---|---|---|
| 회귀 | 연속적인 숫자를 예측 | 집값 예측, 수요 예측 |
| 분류 | 범주형 값을 예측 | 스팸 메일 판별, 손글씨 숫자 인식 |

분류 문제에서는 보통 정확도, 즉 Accuracy를 성능 지표로 사용한다.

---

## 9. KNN

대표적인 비파라메트릭 분류 알고리즘으로 KNN, 즉 K-최근접이웃이 있다.

KNN은 새로운 데이터가 들어오면, 가장 가까운 이웃들의 레이블을 보고 다수결로 분류한다.

```text
새로운 데이터
↓
가까운 K개의 이웃 탐색
↓
다수결로 분류
```

KNN은 직관적이고 단순하지만, 단점도 있다.

- 학습 데이터를 거의 모두 저장해야 함
- 데이터가 많아질수록 메모리 부담이 커짐
- 예측 시 가까운 이웃을 찾아야 하므로 계산 비용이 커질 수 있음

---

## 정리

2장에서 잡아야 할 핵심은 다음과 같다.

1. 머신러닝은 사람이 규칙을 직접 넣는 대신 데이터로부터 패턴을 학습한다.
2. 지도학습은 정답이 있는 데이터로 학습하고, 비지도학습은 정답 없이 구조를 찾는다.
3. 지도학습은 모델 선택, 목표 설정, 파라미터 학습, 예측의 흐름으로 진행된다.
4. 학습 데이터 성능보다 테스트 데이터에서의 일반화 성능이 더 중요하다.
5. 과적합과 과소적합을 피하기 위해 모델 복잡도를 조절해야 한다.
6. 머신러닝 문제는 크게 회귀와 분류로 나눌 수 있다.
