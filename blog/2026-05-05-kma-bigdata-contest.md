## 행사 정보

- **이름 / 주최:** 2026 기상청 빅데이터 콘테스트 / 기상청
- **기간:** 2026.05 ~ 진행중
- **장소·형식:** 온라인 중심 프로젝트 준비
- **팀원:** 팀 프로젝트 진행 예정
- **결과:** 진행중

2026 기상청 빅데이터 콘테스트를 준비하며, **PINN을 활용한 전력설비 인근 화재발생지수 예측** 아이디어를 구체화하고 있습니다. 이번 프로젝트는 단순히 지역별 화재 위험을 예측하는 것이 아니라, 전력설비 주변이라는 구체적인 공간 조건을 기준으로 기상 데이터 기반 위험도를 산출하는 데 초점을 두고 있습니다.

## 우리가 만든 것

- **문제 정의:** 전력설비 주변의 기상 조건이 변화할 때 해당 설비 인근의 화재발생 위험이 어떻게 달라지는지 예측하는 문제
- **솔루션:** 기온, 습도, 풍속, 강수량, 건조 지속일수 등 기상 변수를 전력설비 위치 정보와 결합하고, 데이터 기반 예측 모델에 PINN 기반 물리 제약을 더해 전력설비 인근 화재발생지수를 산출하는 구조
- **데모 / 레포:** 현재 데이터 구조와 모델링 방향을 설계하는 단계

이번 아이디어의 핵심은 예측 단위를 행정구역 전체가 아니라 **전력설비 인근 반경 또는 격자 단위**로 좁히는 것입니다. 이를 통해 “어느 지역이 위험한가”보다 “어떤 전력설비 주변이 현재 조건에서 더 위험한가”를 판단할 수 있는 형태의 결과물을 목표로 합니다.

## 내가 맡은 부분

- 담당 영역: 문제 정의, 데이터 구조 설계, 모델링 방향 구상, PINN 적용 가능성 검토
- 결정한 것 / 만든 것:
  - 지역별 화재 예측이 아니라 전력설비 인근 화재발생지수 예측으로 문제 범위 구체화
  - 기상청 데이터와 전력설비 위치 정보를 결합하는 구조 설계
  - 기온, 습도, 풍속, 강수량, 건조 지속일수 등 주요 입력 변수 정리
  - PINN을 예측 모델의 보조 제약으로 활용하는 방향 설정

## 기술 스택 & 선택 이유

| 분류 | 선택 | 이유 / 대안과의 비교 |
|------|------|----------------------|
| 데이터 | 기상청 기상·기후 데이터 | 화재 위험도에 영향을 주는 기온, 습도, 풍속, 강수량 등을 반영하기 위함 |
| 공간 단위 | 전력설비 인근 반경 또는 격자 | 행정구역 단위보다 실제 시설물 위험 판단에 가까움 |
| 기준 모델 | 머신러닝 기반 예측 모델 | 변수 중요도와 기준 성능을 먼저 확인하기 위한 출발점 |
| 고도화 방향 | PINN | 기상 변수의 시공간적 변화에 물리적 일관성을 반영하기 위함 |
| 산출물 | 위험도 리포트 및 시각화 | 단순 모델 점수보다 현장에서 해석 가능한 결과를 만들기 위함 |

## 잘된 것 / 못한 것

- ✅ 잘된 결정:
  - 문제 범위를 “지역별 화재 예측”에서 “전력설비 인근 화재발생지수 예측”으로 좁힌 점
  - PINN을 화재 발생 자체를 직접 맞히는 만능 모델로 두지 않고, 물리적 일관성을 보조하는 제약 구조로 해석한 점
  - 최종 결과물을 모델 성능 수치가 아니라 위험도 리포트와 시각화 형태로 구상한 점

- ❌ 다시 한다면 바꿀 결정:
  - 처음부터 사용 가능한 데이터 목록과 공간 매칭 가능성을 더 먼저 검토할 필요가 있음
  - PINN 적용 범위를 넓게 잡기보다, 기상 변수의 시공간 변화나 건조도 변화처럼 적용 가능한 부분부터 좁혀야 함

## 배운 점

- 단시간 프로젝트에서는 모델 이름보다 **문제 정의의 구체성**이 더 중요하다는 점을 확인했습니다.
- PINN은 예측 성능을 무조건 높이는 장치가 아니라, 데이터 기반 모델에 물리적 해석 가능성과 제약을 추가하는 방식으로 활용해야 합니다.
- 다음 단계에서는 데이터 확보 가능성, 변수 설계, 기준 모델 성능, PINN 적용 범위를 순서대로 검증하면서 프로젝트를 구체화할 예정입니다.
